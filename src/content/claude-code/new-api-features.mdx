# 🔌 Claude API 신기능

Claude Code CLI 외에도, API 레벨에서 에이전트 개발을 위한 강력한 신기능들이 추가됐어요.

## 🧠 Advisor Tool — 전략가 + 실행자 페어링

빠른 모델(실행자)과 똑똑한 모델(전략가)을 하나의 요청에서 조합하는 기능이에요.

### 핵심 원리

```
사용자 요청
  ↓
[실행 모델: Sonnet] → 복잡한 판단이 필요한 시점 감지
  ↓
[전략 모델: Opus] → 전체 대화를 읽고 400~700 토큰의 전략 지침 생성
  ↓
[실행 모델: Sonnet] → 전략 지침을 따라 작업 계속 (저렴한 비용)
```

### 지원 페어링

| 실행 모델 | 전략 모델 | 용도 |
|----------|----------|------|
| Haiku 4.5 | Opus 4.6 | 비용 최적화 (가장 저렴) |
| Sonnet 4.6 | Opus 4.6 | 균형 (성능 대비 비용) |
| Opus 4.6 | Opus 4.6 | 최고 품질 (자기 자문) |

### 어떤 상황에서 유용한가?

- 코딩 에이전트가 수십 단계를 반복할 때 — 매번 Opus를 쓰면 비싸고, Sonnet만 쓰면 전략 판단이 약함
- 복잡한 분기 판단(아키텍처 결정, 디버깅 방향)에서만 Opus가 개입

## 🏗️ Managed Agents — 완전 관리형 에이전트

에이전트 루프를 직접 구축하지 않아도 되는 관리형 인프라예요.

### 기존 방식 vs Managed Agents

```
기존: 에이전트 루프 직접 구현
  while (not done):
    response = call_api()
    result = execute_tool(response)
    # 에러 처리, 컨텍스트 관리, 타임아웃... 직접 구현

Managed Agents: API 한 번 호출
  session = create_session(agent, environment)
  → 컨테이너, 도구, 캐싱, 압축 모두 제공
  → SSE로 실시간 결과 스트리밍
```

### 4가지 핵심 개념

| 개념 | 설명 |
|------|------|
| **Agent** | 모델 + 시스템 프롬프트 + 도구 정의 |
| **Environment** | 보안 컨테이너 템플릿 (Bash, 파일, 네트워크 설정) |
| **Session** | Agent + Environment의 실행 인스턴스 |
| **Event** | 세션에서 발생하는 메시지·결과 스트림 |

### 내장 도구

Bash, 파일 읽기/쓰기, 웹 검색/패치, MCP 연결 — 별도 구현 없이 바로 사용 가능.

## 🔗 MCP 강화 — 더 넓고 빠르게

MCP(Model Context Protocol)가 대폭 개선됐어요:

| 기능 | 설명 |
|------|------|
| **MCP Connector** | 로컬 서버 없이 원격 MCP 서버에 직접 연결 |
| **OAuth 인증** | MCP 서버에 OAuth 플로우로 안전하게 인증 |
| **동적 주입** | 세션 중간에 MCP 서버를 재시작 없이 추가 |
| **도구 지연 로딩** | 많은 MCP 서버를 연결해도 컨텍스트 사용량 95% 절감 |
| **대용량 결과** | 도구 결과를 최대 500K 문자까지 수신 가능 |
| **논블로킹 연결** | 5초 타임아웃으로 MCP 서버 연결 실패가 전체를 막지 않음 |

### MCP Connector의 의미

```
이전: 모든 MCP 서버를 로컬에서 실행
  npx @modelcontextprotocol/server-github  ← 로컬 프로세스

이후: 원격 서버에 직접 연결
  API 요청에 MCP 서버 URL만 지정 → 로컬 설치 불필요
```

## 💬 시나리오 예시

### 예시 1: 비용 효율적 코딩 에이전트

**상황**: 100단계짜리 코드 리팩토링 에이전트를 만들어야 한다.

> ❌ **Before (단일 모델)**
> 전체를 Opus로 실행하면 비용이 폭발하고, Sonnet으로만 하면 복잡한 아키텍처 판단에서 실수한다.

> ✅ **After (Advisor Tool)**
> Sonnet이 실행하다가 "이 모듈 구조를 어떻게 잡지?" 같은 전략적 판단 시점에만 Opus가 개입한다. 비용은 Sonnet 수준, 품질은 Opus에 근접한다.

### 예시 2: 에이전트 인프라 구축

**상황**: 고객 문의를 자동 처리하는 AI 에이전트를 만들어야 한다.

> ❌ **Before (직접 구현)**
> 에이전트 루프, 도구 호출, 에러 처리, 컨텍스트 압축, 컨테이너 격리를 모두 직접 코딩한다. 수주가 걸린다.

> ✅ **After (Managed Agents)**
> Agent(역할 정의) + Environment(도구 설정)만 선언하면 나머지는 관리형 인프라가 처리한다. 수시간 내 프로토타입이 가능하다.

### 예시 3: MCP 서버 관리 간소화

**상황**: GitHub + Slack + DB 3개 MCP 서버를 연결해야 한다.

> ❌ **Before (로컬 프로세스)**
> 3개 서버를 각각 `npx`로 로컬에서 실행해야 한다. 프로세스 관리, 환경변수, 포트 충돌을 신경 써야 한다.

> ✅ **After (MCP Connector + 동적 주입)**
> API 요청에 원격 MCP 서버 URL만 지정한다. 세션 중간에 서버를 추가/제거할 수도 있고, 연결 실패가 전체를 막지 않는다.

## 📖 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **Advisor Tool** | 실행 모델 + 전략 모델을 하나의 요청에서 조합하는 기능 | Sonnet(실행) + Opus(전략) |
| **Managed Agents** | 에이전트 루프·컨테이너·도구를 관리형으로 제공하는 API | Agent + Environment + Session |
| **MCP Connector** | 로컬 서버 없이 원격 MCP 서버에 직접 연결하는 기능 | API에 URL만 지정 |
| **SSE** | Server-Sent Events, 서버에서 클라이언트로 실시간 스트리밍 | 에이전트 실행 결과 실시간 수신 |
| **도구 지연 로딩** | MCP 도구를 필요할 때만 로드해 컨텍스트를 절약하는 기법 | 컨텍스트 사용량 95% 절감 |
| **OAuth** | 외부 서비스에 안전하게 인증하는 표준 프로토콜 | MCP 서버 인증 |

## 📌 핵심 정리

- **Advisor Tool**: 저렴한 모델이 실행하고, 전략적 판단에서만 고급 모델이 개입
- **Managed Agents**: 에이전트 인프라를 직접 구축하지 않고 API로 관리형 실행
- **MCP 강화**: 원격 연결, OAuth, 동적 주입, 지연 로딩으로 확장성 향상
- 이 기능들은 Claude Code CLI가 아닌 **Anthropic API** 레벨의 기능이에요

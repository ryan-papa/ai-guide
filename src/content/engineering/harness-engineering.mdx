# 🎛️ 하네스 엔지니어링

하네스 엔지니어링은 AI 에이전트를 실제 환경에서 **안전하고 효과적으로 운영**하기 위한 통제 시스템을 설계하는 기술이에요.

## 🐎 하네스란?

하네스(Harness)는 원래 말의 고삐와 안장을 뜻해요. AI에서의 하네스는 에이전트의 **행동을 통제하고 방향을 잡아주는 프레임워크**예요.

```
[사용자] → [하네스] → [에이전트] → [도구/환경]
              ↑                         |
              └── 모니터링 ←────────────┘
```

에이전트가 아무리 똑똑해도, 하네스 없이는 안전하게 운영할 수 없어요.

## 🏗️ 하네스의 핵심 구성 요소

### 1. 권한 관리 (Permission Layer)
에이전트가 할 수 있는 것과 없는 것을 정의해요:

```yaml
permissions:
  file_system:
    read: true
    write: "src/ 하위만"
    delete: false
  shell:
    allowed_commands: ["npm test", "npm run build"]
    blocked_commands: ["rm -rf", "sudo"]
  network:
    allowed_domains: ["api.github.com"]
```

### 2. 승인 게이트 (Approval Gate)
위험도에 따라 자동/수동 승인을 나눠요:

| 위험도 | 예시 | 처리 |
|--------|------|------|
| 낮음 | 파일 읽기, 검색 | 자동 승인 |
| 중간 | 파일 수정, 코드 실행 | 알림 후 자동 |
| 높음 | 파일 삭제, DB 변경 | 사람 승인 필요 |
| 치명적 | 프로덕션 배포, 결제 | 다중 승인 필요 |

### 3. 실행 샌드박스 (Sandbox)
에이전트의 실행 환경을 격리해요:
- **컨테이너 격리**: Docker 등으로 시스템 영향 차단
- **네트워크 제한**: 허용된 도메인만 접근 가능
- **리소스 제한**: CPU, 메모리, 실행 시간 제한

### 4. 모니터링과 로깅
모든 행동을 추적하고 기록해요:

```json
{
  "timestamp": "2026-04-11T10:30:00Z",
  "agent_id": "coding-agent-01",
  "action": "edit_file",
  "target": "src/auth/login.ts",
  "reasoning": "인증 로직의 타입 에러 수정",
  "status": "approved",
  "duration_ms": 1200
}
```

## 🔧 실전 하네스 패턴

### 패턴 1: 가드레일 패턴
에이전트의 출력을 필터링하는 별도의 검증 레이어를 두는 방식이에요:

```
에이전트 출력 → [가드레일 검증]
  ├→ ✅ 안전 → 실행
  ├→ ⚠️ 의심 → 사람에게 확인 요청
  └→ ❌ 위험 → 차단 + 알림
```

### 패턴 2: 체크포인트 패턴
중요한 작업 전에 상태를 저장하고, 문제 발생 시 복원해요:

```
[작업 시작]
  → 체크포인트 저장 (git stash, 스냅샷)
  → 작업 실행
  → 결과 검증
    ├→ 성공 → 체크포인트 삭제
    └→ 실패 → 체크포인트로 복원
```

### 패턴 3: 에스컬레이션 패턴
에이전트가 판단하기 어려운 상황을 상위로 넘겨요:

```
에이전트: "이 파일을 수정해야 하는데,
  테스트가 없어서 안전성을 보장할 수 없어요.
  사람이 확인해 주시겠어요?"
```

## 📊 Claude Code의 하네스

Claude Code는 하네스 엔지니어링의 좋은 실례예요:

- **권한**: `settings.json`에서 허용할 도구와 명령을 설정
- **승인**: 파일 수정, 셸 명령 실행 전 사용자 확인
- **맥락**: `CLAUDE.md`로 프로젝트 규칙을 지속적으로 전달
- **hooks**: 특정 이벤트에 자동 실행되는 스크립트 설정

```json
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep"],
    "deny": ["Bash(rm -rf *)"]
  }
}
```

## ⚡ 하네스 설계 체크리스트

좋은 하네스를 설계하기 위한 체크리스트예요:

- [ ] 에이전트의 **최소 필요 권한**을 정의했는가?
- [ ] **위험한 작업**에 대한 승인 프로세스가 있는가?
- [ ] **실행 환경**이 격리되어 있는가?
- [ ] 모든 행동이 **로깅**되고 있는가?
- [ ] **비용 한도**가 설정되어 있는가?
- [ ] **실패 시 복구** 메커니즘이 있는가?
- [ ] **에스컬레이션** 경로가 정의되어 있는가?

## 🔮 미래 전망

하네스 엔지니어링은 AI 에이전트가 더 자율적이 될수록 더 중요해질 거예요:

- **적응형 하네스**: 에이전트의 신뢰도에 따라 자동으로 권한 조정
- **멀티 에이전트 하네스**: 여러 에이전트 간의 상호작용을 관리
- **규제 준수 하네스**: 법적 요구사항을 자동으로 반영

## 📌 핵심 정리

- 하네스는 AI 에이전트를 **안전하게 통제**하는 프레임워크예요
- 권한 관리, 승인 게이트, 샌드박스, 모니터링이 핵심 구성 요소예요
- 가드레일, 체크포인트, 에스컬레이션 패턴을 활용하세요
- 에이전트가 강력해질수록 **하네스의 중요성**도 커져요

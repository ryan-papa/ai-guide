# ⚡ 트랜스포머의 등장

2017년 구글이 발표한 논문 **"Attention Is All You Need"**는 AI 역사에서 가장 영향력 있는 논문 중 하나예요. 이 논문에서 소개된 **Transformer**가 오늘날 AI의 핵심 아키텍처가 되었어요.

## 🤔 트랜스포머 이전의 문제

기존 RNN(순환 신경망)에는 심각한 한계가 있었어요:

| 문제 | 설명 |
|------|------|
| **순차 처리** | 단어를 하나씩 처리해서 느림 |
| **장기 의존성** | 먼 거리의 단어 관계를 잊어버림 |
| **병렬화 불가** | GPU의 장점을 활용하기 어려움 |

예를 들어 "서울에서 태어나 부산에서 자란 그는 ... (200단어) ... 고향을 그리워했다"라는 문장에서 "고향"이 "서울"인지 "부산"인지 RNN은 혼동할 수 있어요.

## 💡 어텐션 메커니즘

트랜스포머의 핵심은 **셀프 어텐션(Self-Attention)**이에요.

### 핵심 아이디어
문장의 모든 단어가 다른 모든 단어와의 관계를 동시에 계산해요:

```
"고양이가 매트 위에 앉아 있다"

고양이 → 앉아 (강한 연결: 누가 앉아 있는지)
매트   → 위에 (강한 연결: 어디에 있는지)
고양이 → 매트 (중간 연결: 어디에 앉아 있는지)
```

### QKV (Query, Key, Value)
어텐션은 세 가지 요소로 작동해요:

- **Query(질문)**: "나와 관련된 단어가 뭐지?"
- **Key(열쇠)**: "나는 이런 단어야"
- **Value(값)**: "나의 실제 정보는 이거야"

도서관에 비유하면:
- Query는 "AI에 대한 책을 찾고 있어요"라는 요청
- Key는 각 책의 제목과 태그
- Value는 책의 실제 내용

## 🏗️ 트랜스포머의 구조

```
입력 → [인코더] → 중간 표현 → [디코더] → 출력

인코더: 입력을 이해하는 부분
디코더: 출력을 생성하는 부분
```

각 인코더/디코더는 여러 층으로 쌓여 있고, 각 층에는:
1. **멀티헤드 어텐션** — 여러 관점에서 동시에 관계를 파악
2. **피드포워드 네트워크** — 추출된 정보를 가공
3. **잔차 연결 + 정규화** — 깊은 학습을 안정적으로

## 🌳 트랜스포머에서 갈라진 세 갈래

### 인코더만 사용: BERT 계열
문장을 **양방향**으로 이해하는 데 특화돼요.
- 텍스트 분류, 감성 분석, 질의응답
- 대표: BERT, RoBERTa

### 디코더만 사용: GPT 계열
텍스트를 **생성**하는 데 특화돼요.
- 대화, 글쓰기, 코딩
- 대표: GPT-4, Claude, Llama

### 인코더-디코더: T5 계열
입력을 이해하고 다른 형태로 **변환**하는 데 특화돼요.
- 번역, 요약
- 대표: T5, BART

## 📊 트랜스포머가 바꾼 것들

트랜스포머 이전과 이후의 변화:

| 영역 | 이전 | 이후 |
|------|------|------|
| 자연어 처리 | 작업별 전문 모델 | 하나의 범용 모델 |
| 이미지 처리 | CNN 일색 | Vision Transformer(ViT) |
| 코드 생성 | 규칙 기반 | AI 코드 어시스턴트 |
| 학습 방식 | 소규모, 작업별 | 대규모 사전학습 |

## 🔢 스케일링 법칙

트랜스포머에서 발견된 놀라운 법칙이에요:

> **모델 크기 + 데이터 양 + 컴퓨팅이 증가하면, 성능이 예측 가능하게 향상된다**

이 발견 덕분에 기업들은 "더 크게, 더 많이" 투자하게 됐고, GPT-3 → GPT-4로의 도약이 가능했어요.

## 🔑 왜 트랜스포머가 성공했나?

1. **병렬 처리** — 모든 단어를 동시에 처리해서 GPU 활용 극대화
2. **장거리 의존성** — 멀리 떨어진 단어 간의 관계도 정확히 포착
3. **범용성** — 텍스트, 이미지, 음성, 코드 등 다양한 분야에 적용 가능
4. **확장성** — 모델을 키울수록 성능이 안정적으로 향상

## 💬 시나리오 예시

### 예시 1: 문서 요약

**상황**: 100페이지 연구 보고서를 1페이지로 요약해야 한다.

> ❌ **Before (RNN 기반)**
> 긴 문서를 순차 처리하다가 앞부분 내용을 잊어버린다. 결론만 반영된 편향적 요약이 나온다.

> ✅ **After (Transformer 기반)**
> 어텐션으로 문서 전체를 동시에 파악해, 서론·본론·결론을 균형 있게 요약한다.

### 예시 2: 코드 자동 완성

**상황**: 개발자가 함수를 작성하는 중이다.

> ❌ **Before (패턴 매칭)**
> 최근 입력한 몇 줄만 보고 단순 키워드를 제안한다. 프로젝트 전체 맥락을 모른다.

> ✅ **After (GPT 계열 모델)**
> 파일 전체와 프로젝트 구조를 이해해서 논리에 맞는 다음 코드 블록을 생성한다.

### 예시 3: 감성 분석

**상황**: 제품 리뷰 10만 건의 긍정/부정을 분류해야 한다.

> ❌ **Before (키워드 기반)**
> "좋다"=긍정, "나쁘다"=부정으로 분류한다. "가격 대비 나쁘지 않다"를 부정으로 오분류한다.

> ✅ **After (BERT 계열)**
> 문맥을 양방향으로 이해해 "나쁘지 않다"가 긍정 표현임을 정확히 파악한다.

## 📖 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **어텐션** | 입력의 어떤 부분에 집중할지 가중치를 부여하는 메커니즘 | "고양이가 앉아 있다"에서 주어-동사 연결 |
| **QKV** | Query·Key·Value로 단어 간 관련도를 계산하는 구조 | 도서관에서 질문→제목 매칭→내용 반환 |
| **인코더** | 입력을 이해하고 내부 표현으로 변환하는 부분 | BERT (텍스트 분류, 감성 분석) |
| **디코더** | 내부 표현에서 출력을 생성하는 부분 | GPT (텍스트 생성, 대화) |
| **스케일링 법칙** | 모델·데이터·컴퓨팅 증가 시 성능이 예측 가능하게 향상되는 법칙 | GPT-3 → GPT-4 도약의 근거 |
| **멀티헤드 어텐션** | 여러 관점에서 동시에 어텐션을 수행하는 기법 | 문법·의미·감정을 동시에 파악 |

## 📌 핵심 정리

- 트랜스포머는 **어텐션 메커니즘**으로 기존 RNN의 한계를 극복했어요
- **QKV 어텐션**으로 단어 간 관계를 동시에 파악해요
- BERT(인코더), GPT(디코더), T5(인코더-디코더) 세 갈래로 발전했어요
- **스케일링 법칙**이 현대 AI 경쟁의 원동력이에요

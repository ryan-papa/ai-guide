# 🧩 컨텍스트 엔지니어링

컨텍스트 엔지니어링은 AI에게 전달하는 맥락(context)을 체계적으로 설계하고 관리하는 기술이에요. 프롬프트 엔지니어링의 자연스러운 다음 단계예요.

> 💡 **비유로 이해하기**: 프롬프트 엔지니어링이 **"질문을 잘 하는 기술"**이라면, 컨텍스트 엔지니어링은 **"필요한 참고 자료를 잘 준비하는 기술"**이에요. 시험을 볼 때 좋은 질문을 만드는 것도 중요하지만, 교과서·노트·기출문제를 잘 정리해서 가져가는 것이 성적에 더 큰 영향을 미치는 것과 같아요.

## 🤔 프롬프트 vs 컨텍스트

프롬프트 엔지니어링이 "어떻게 물어볼까?"라면, 컨텍스트 엔지니어링은 **"어떤 정보를 함께 줄까?"**에요.

```
프롬프트 엔지니어링:
  "이 코드를 리팩토링해 줘" (질문 방식에 집중)

컨텍스트 엔지니어링:
  시스템 프롬프트 + 코딩 컨벤션 문서 + 관련 코드 파일들
  + 아키텍처 문서 + "이 코드를 리팩토링해 줘"
  (주변 정보 설계에 집중)
```

## 📦 컨텍스트의 네 가지 층

### 1. 시스템 프롬프트 (System Prompt)
AI의 행동 방식과 규칙을 정의해요:
```
"당신은 TypeScript 전문가입니다.
모든 코드에 타입을 명시하고,
에러 처리를 빠뜨리지 마세요."
```

### 2. 참조 자료 (Reference)
AI가 참고해야 할 문서나 데이터를 제공해요:
- API 문서, 코딩 컨벤션, 디자인 시스템 가이드
- 기존 코드베이스의 패턴

### 3. 대화 기록 (Conversation History)
이전 대화의 맥락을 유지해요:
- 멀티턴 대화에서 앞선 내용을 기억
- 이전 결정사항과 합의 내용

### 4. 작업 맥락 (Task Context)
현재 수행 중인 작업의 구체적인 상황이에요:
- 현재 파일, 에러 메시지, 실행 환경

## 🛠️ 실전 컨텍스트 설계

### CLAUDE.md 활용
Claude Code에서는 프로젝트 루트에 `CLAUDE.md` 파일을 두어 지속적인 컨텍스트를 제공해요:

```markdown
# 프로젝트 규칙
- TypeScript strict 모드 사용
- 컴포넌트는 함수형으로 작성
- 테스트 커버리지 80% 이상 유지
- 커밋 메시지는 Conventional Commits 형식
```

### RAG (검색 증강 생성)
필요한 정보를 실시간으로 검색해서 컨텍스트에 추가하는 기법이에요:

```
사용자 질문: "우리 프로젝트의 인증 로직을 수정해 줘"

RAG 시스템:
1. 질문에서 "인증"이라는 키워드 추출
2. 코드베이스에서 인증 관련 파일 검색
3. auth.ts, middleware.ts 등을 컨텍스트에 추가
4. AI가 실제 코드를 참고하여 정확한 수정안 제시
```

> 📰 **실제 사례**: 고객 상담 챗봇을 만든다고 생각해보세요. AI에게 회사 제품 정보를 모두 외우게 할 수 없어요(학습 데이터에 없으니까). RAG를 사용하면 고객이 "배송 정책이 뭐예요?"라고 물을 때, 실시간으로 최신 배송 정책 문서를 검색해서 AI에게 전달해요. AI는 그 문서를 참고해 정확한 답을 해줘요. **ChatGPT의 "검색" 기능, Claude의 "파일 첨부" 기능**이 모두 이 원리예요.

### 컨텍스트 윈도우 관리
AI의 맥락 창은 무한하지 않아요. 효율적으로 관리해야 해요:

| 전략 | 설명 |
|------|------|
| **우선순위** | 가장 관련 있는 정보를 앞에 배치 |
| **요약** | 긴 대화는 핵심만 요약해서 유지 |
| **선택적 포함** | 관련 없는 정보는 제외 |
| **청크 분할** | 긴 문서를 의미 단위로 나눠서 필요한 부분만 포함 |

## 💡 컨텍스트 설계 패턴

### 패턴 1: 레이어드 컨텍스트
```
[글로벌 규칙]     → 항상 적용되는 기본 규칙
  [프로젝트 규칙]  → 프로젝트별 설정
    [작업 규칙]    → 현재 작업에 특화된 지시
```

### 패턴 2: 동적 컨텍스트
상황에 따라 필요한 정보를 동적으로 로딩해요:
- 코드 수정 시 → 관련 파일과 테스트 파일 로딩
- 버그 수정 시 → 에러 로그와 스택 트레이스 로딩
- 문서 작성 시 → 기존 문서 스타일과 구조 로딩

### 패턴 3: MCP (Model Context Protocol)
AI와 외부 시스템을 표준화된 프로토콜로 연결하는 기법이에요:

```
[MCP 서버]
- 파일 시스템 → 프로젝트 파일 읽기/쓰기
- 데이터베이스 → 스키마·데이터 조회
- 외부 API → Slack, GitHub, Jira 연동
- 브라우저 → 웹 페이지 탐색·스크린샷
```

MCP를 사용하면 AI가 필요한 컨텍스트를 **도구를 통해 동적으로 가져올** 수 있어요. 프롬프트에 모든 정보를 직접 넣는 대신, AI가 필요할 때 적절한 MCP 서버에 요청하는 방식이에요.

### 패턴 4: 메모리 패턴
중요한 결정사항을 별도로 저장하고 재사용해요:
```
[이전 세션 메모]
- DB 스키마 변경은 마이그레이션 파일로 관리하기로 함
- API 응답 형식은 JSON:API 스펙을 따르기로 함
```

## ⚠️ 컨텍스트 오버로드 주의

너무 많은 컨텍스트는 오히려 역효과를 낼 수 있어요:

- **관련 없는 정보**가 많으면 AI가 혼란스러워해요
- **중복된 지시**가 있으면 모순이 생길 수 있어요
- **맥락 창 초과** 시 중요한 정보가 잘려나갈 수 있어요

핵심은 **"적절한 정보를, 적절한 양으로, 적절한 시점에"** 제공하는 것이에요.

> 🎯 **실무 팁 — 컨텍스트 우선순위 정하기**:
> 1. **가장 중요**: 현재 작업과 직접 관련된 파일/데이터
> 2. **중요**: 프로젝트 규칙, 코딩 컨벤션
> 3. **참고용**: 관련 문서, 이전 대화 요약
> 4. **제외**: 관련 없는 파일, 중복 정보
>
> 맥락 창의 **앞부분에 있는 정보**가 AI에게 더 큰 영향을 줘요. 가장 중요한 정보를 맨 앞에 배치하세요.

## 💬 시나리오 예시

### 예시 1: 코드 리팩토링 요청

**상황**: AI에게 인증 모듈 리팩토링을 요청한다.

> ❌ **Before (맥락 없이)**
> "auth.ts를 리팩토링해 줘." → AI가 프로젝트 컨벤션을 모르고 임의 스타일로 변경한다.

> ✅ **After (컨텍스트 설계)**
> CLAUDE.md에 코딩 컨벤션 + 관련 파일(auth.ts, middleware.ts, types.ts)을 함께 제공한다. AI가 프로젝트 패턴에 맞춰 리팩토링한다.

### 예시 2: RAG 활용

**상황**: 사내 위키에 있는 정보를 기반으로 AI가 답변해야 한다.

> ❌ **Before (정적 프롬프트)**
> AI에게 사내 규정을 수동으로 복붙한다. 규정이 변경되면 프롬프트도 매번 수정해야 한다.

> ✅ **After (RAG 시스템)**
> 질문이 들어오면 관련 위키 문서를 자동 검색해 컨텍스트에 추가한다. 규정이 변경되면 별도 수정 없이 최신 정보를 반영한다.

### 예시 3: 맥락 창 관리

**상황**: 10만 줄 코드베이스에서 버그를 수정해야 한다.

> ❌ **Before (전체 투입)**
> 코드 전체를 AI에게 넣으려 한다. 맥락 창을 초과하거나 핵심이 묻힌다.

> ✅ **After (선택적 컨텍스트)**
> 에러 로그 → 관련 파일 3~4개만 검색해서 제공한다. AI가 집중해서 정확한 수정안을 제시한다.

## 📖 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **컨텍스트** | AI에게 전달하는 배경 정보·참조 자료의 총체 | 시스템 프롬프트, 코드, 문서 |
| **맥락 창(Context Window)** | AI가 한 번에 처리할 수 있는 토큰 수 상한 | 128K, 200K, 1M 토큰 |
| **RAG** | 질문에 관련된 문서를 검색해 컨텍스트에 추가하는 기법 | 사내 위키 자동 검색 |
| **시스템 프롬프트** | AI의 행동 규칙·역할을 정의하는 초기 설정 | "TypeScript 전문가입니다" |
| **CLAUDE.md** | Claude Code에서 프로젝트 규칙을 지속 전달하는 파일 | 코딩 컨벤션, 아키텍처 규칙 |
| **MCP** | AI와 외부 시스템을 표준 프로토콜로 연결하는 기술 | DB, Slack, GitHub 연동 |
| **청크 분할** | 긴 문서를 의미 단위로 나눠 필요한 부분만 제공하는 기법 | 10만 줄 코드 → 관련 파일 3개 |

## 📌 핵심 정리

- 컨텍스트 엔지니어링은 **"어떤 정보를 AI에게 줄 것인가"**를 설계하는 기술이에요
- 시스템 프롬프트, 참조 자료, 대화 기록, 작업 맥락의 **4개 층**으로 구성해요
- **RAG**를 통해 필요한 정보를 동적으로 제공할 수 있어요
- 맥락 창을 **효율적으로 관리**하는 것이 핵심이에요

> 🌟 **초보자 팁**: 컨텍스트 엔지니어링을 어렵게 생각하지 마세요. 가장 간단한 시작은 AI에게 질문할 때 **관련 파일을 함께 첨부하는 것**이에요. Claude에서 파일을 드래그앤드롭하거나, ChatGPT에서 파일을 업로드하는 것만으로도 컨텍스트 엔지니어링을 하고 있는 거예요!

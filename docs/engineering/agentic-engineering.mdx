# 🤖 에이전틱 엔지니어링

에이전틱 엔지니어링은 AI가 단순 응답을 넘어 **스스로 계획하고, 도구를 사용하고, 작업을 완수**하도록 시스템을 설계하는 기술이에요.

> 💡 **비유로 이해하기**: 챗봇은 **네비게이션**이에요 — "여기서 좌회전하세요"라고 안내만 하죠. 에이전트는 **자율주행차**예요 — 목적지만 알려주면 스스로 경로를 계획하고, 핸들을 돌리고, 장애물을 피하며 도착해요. 프롬프트 → 컨텍스트 → 에이전트 순으로 AI의 자율성이 높아지는 거예요.

## 💬 챗봇 vs 에이전트

| 구분 | 챗봇 | 에이전트 |
|------|------|---------|
| 작동 방식 | 질문 → 답변 | 목표 → 계획 → 실행 → 검증 |
| 도구 사용 | ❌ | ✅ 파일, API, 검색 등 |
| 자율성 | 낮음 | 높음 |
| 반복 | 1회 응답 | 목표 달성까지 반복 |

```
챗봇: "이 버그를 어떻게 고칠까요?" → 설명을 해줌

에이전트: "이 버그를 고쳐줘"
  → 코드를 분석
  → 원인을 파악
  → 수정 코드를 작성
  → 테스트를 실행
  → 결과를 보고
```

## 🔄 에이전트의 핵심 루프

에이전트는 **관찰-사고-행동(Observe-Think-Act)** 루프로 작동해요:

```
1. 관찰 (Observe)
   → 현재 상태를 파악해요
   → 파일 읽기, 에러 확인, 검색 결과 확인

2. 사고 (Think)
   → 다음에 무엇을 할지 계획해요
   → "이 에러는 타입 불일치 문제니까..."

3. 행동 (Act)
   → 도구를 사용해 실행해요
   → 파일 수정, 명령어 실행, API 호출

4. 반복
   → 목표가 달성될 때까지 1~3을 반복
```

## 🧰 도구 사용 (Tool Use)

에이전트의 핵심 능력은 **도구를 적절하게 선택하고 사용**하는 것이에요:

### 도구의 종류
- **파일 시스템**: 파일 읽기, 쓰기, 검색
- **코드 실행**: 셸 명령, 스크립트 실행
- **웹 검색**: 최신 정보 조회
- **API 호출**: 외부 서비스 연동
- **데이터베이스**: 쿼리 실행, 데이터 조회

### 도구 정의 예시
```json
{
  "name": "read_file",
  "description": "파일의 내용을 읽어옵니다",
  "parameters": {
    "file_path": {
      "type": "string",
      "description": "읽을 파일의 경로"
    }
  }
}
```

AI는 이 정의를 보고, 필요할 때 적절한 도구를 선택해서 호출해요.

> 🎯 **실무 팁 — 좋은 도구 설명 작성법**: 에이전트가 도구를 잘 사용하려면 `description`이 핵심이에요. AI는 이 설명을 읽고 "지금 이 도구를 써야 하나?"를 판단해요. 모호한 설명("파일 관련 작업")보다 구체적인 설명("지정된 경로의 파일 내용을 텍스트로 읽어 반환합니다")이 훨씬 좋은 결과를 만들어요.

## 🏗️ 에이전트 설계 패턴

### 패턴 1: 단일 에이전트
하나의 에이전트가 모든 작업을 처리해요. 간단한 작업에 적합해요.

### 패턴 2: 오케스트레이터 패턴
메인 에이전트가 서브에이전트에게 작업을 위임해요:
```
오케스트레이터: "로그인 기능을 구현해야 해"
  → 코드 에이전트: "인증 로직 작성"
  → 테스트 에이전트: "테스트 코드 작성"
  → 리뷰 에이전트: "코드 품질 검증"
```

### 패턴 3: 파이프라인 패턴
작업을 순서대로 연결해요:
```
분석 → 계획 → 구현 → 테스트 → 배포
```

## ⚠️ 에이전트의 위험과 안전장치

에이전트에게 자율성을 줄수록 **위험**도 커져요:

### 주요 위험
- **무한 루프**: 같은 작업을 끝없이 반복
- **잘못된 실행**: 파일을 잘못 삭제하거나 수정
- **비용 폭발**: API 호출이 과도하게 발생
- **보안 문제**: 민감한 데이터에 대한 접근

### 안전장치 설계
```
1. 권한 제한: 삭제, 배포 등 위험한 작업은 사람 승인 필요
2. 실행 횟수 제한: 최대 반복 횟수 설정
3. 비용 한도: API 호출 비용 상한선 설정
4. 샌드박스: 격리된 환경에서 실행
5. 로깅: 모든 행동을 기록하고 추적
```

## 📊 실제 에이전트 사례 (2026년 기준)

### Claude Code
터미널·데스크톱·웹·IDE에서 코드를 직접 읽고, 수정하고, 실행하는 코딩 에이전트예요. 서브에이전트 위임, MCP 서버 연동, hooks를 통한 하네스 제어를 지원해요.

### GitHub Copilot Coding Agent
이슈를 분석하고 자율적으로 브랜치 생성 → 코드 수정 → PR 생성까지 수행해요.

### Devin / OpenAI Codex
자율적으로 개발 작업을 수행하는 AI 소프트웨어 엔지니어예요. 브라우저·터미널·에디터를 동시에 사용해요.

## 🔑 에이전트 설계의 핵심 원칙

1. **최소 권한** — 필요한 만큼만 권한을 부여하세요
2. **인간 개입 지점** — 중요한 결정에는 사람의 확인을 받으세요
3. **실패 복구** — 에러 발생 시 안전하게 복구할 방법을 마련하세요
4. **투명성** — AI가 무엇을 왜 하는지 항상 확인할 수 있게 하세요
5. **점진적 자율성** — 신뢰가 쌓일수록 단계적으로 자율성을 높이세요

> 📰 **2026년 에이전트 현황**: AI 에이전트는 실무에 본격 도입됐어요. Claude Code는 터미널·데스크톱·웹·IDE에서 코드를 직접 수정하고, GitHub Copilot Coding Agent는 이슈에서 PR까지 자율 수행하며, Cursor/Windsurf 등 에이전틱 IDE가 보편화됐어요. 하지만 "완전 자율"보다 **"사람과 협업하는 에이전트"**가 실무에서 더 효과적이라는 것이 업계 공감대예요.

## 💬 시나리오 예시

### 예시 1: 버그 수정

**상황**: 프로덕션에서 로그인 실패 에러가 발생했다.

> ❌ **Before (챗봇)**
> "이 에러 어떻게 고쳐?" → "JWT 만료 문제일 수 있습니다"라고 설명만 한다. 개발자가 직접 코드를 찾아 수정해야 한다.

> ✅ **After (에이전트)**
> 에러 로그 분석 → 관련 파일(auth.ts) 탐색 → 토큰 갱신 로직 누락 발견 → 코드 수정 → 테스트 실행 → 결과 보고까지 자동 수행한다.

### 예시 2: 멀티 에이전트 협업

**상황**: 새 기능을 구현하고 배포까지 해야 한다.

> ❌ **Before (단일 처리)**
> 하나의 AI에게 설계·구현·테스트·배포를 모두 맡긴다. 범위가 넓어 품질이 떨어진다.

> ✅ **After (오케스트레이터 패턴)**
> 메인 에이전트가 코드 에이전트(구현), 테스트 에이전트(검증), 리뷰 에이전트(품질)에게 각각 위임한다. 전문화된 결과를 통합한다.

### 예시 3: 안전장치의 중요성

**상황**: 에이전트가 DB 마이그레이션을 수행한다.

> ❌ **Before (무제한 권한)**
> 에이전트가 프로덕션 DB에 직접 ALTER TABLE을 실행한다. 실수 시 데이터가 유실된다.

> ✅ **After (승인 게이트)**
> 에이전트가 마이그레이션 SQL을 생성하고 "프로덕션 DB 변경이 필요합니다"라고 사람에게 승인을 요청한다. 승인 후에만 실행된다.

## 📖 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **에이전트** | 도구를 사용해 자율적으로 작업을 수행하는 AI | Claude Code, Devin |
| **도구 사용(Tool Use)** | AI가 외부 기능(파일, API 등)을 호출하는 능력 | 파일 읽기, 셸 명령 실행 |
| **오케스트레이터** | 서브에이전트에게 작업을 위임하는 메인 에이전트 | 코드·테스트·리뷰 에이전트 관리 |
| **OTA 루프** | 관찰(Observe)-사고(Think)-행동(Act) 반복 구조 | 에러 확인 → 분석 → 수정 |
| **샌드박스** | 에이전트의 실행 환경을 격리하는 보안 장치 | Docker 컨테이너, 가상 환경 |

## 📌 핵심 정리

- 에이전트는 **관찰-사고-행동** 루프로 자율적으로 작업을 수행해요
- **도구 사용**이 에이전트의 핵심 능력이에요
- 오케스트레이터, 파이프라인 등 다양한 **설계 패턴**이 있어요
- 자율성이 높을수록 **안전장치**가 중요해요

> 🌟 **초보자 팁**: 에이전트를 직접 만들 필요 없이, 이미 잘 만들어진 에이전트를 **사용해보는 것**부터 시작하세요. Claude Code를 터미널에서 써보거나, ChatGPT의 "Browse" 기능을 활용해보면 에이전트가 어떻게 작동하는지 체감할 수 있어요. 사용 경험이 쌓이면 설계도 자연스럽게 이해돼요.

## 프롬프트 엔지니어링

프롬프트 엔지니어링은 AI에게 원하는 결과를 얻기 위해 입력(프롬프트)을 효과적으로 설계하는 기술이에요.

> 💡 **비유로 이해하기**: 프롬프트는 AI에게 보내는 **주문서**와 같아요. 카페에서 "커피 주세요"라고 하면 아메리카노가 나올 수도, 라떼가 나올 수도 있어요. "아이스 아메리카노, 톨 사이즈, 샷 추가"처럼 구체적으로 주문할수록 원하는 결과를 정확히 받을 수 있어요.

### 왜 프롬프트가 중요한가요?

같은 AI에게 같은 질문을 해도, **어떻게 물어보느냐**에 따라 결과가 완전히 달라져요:

```
❌ 나쁜 프롬프트:
"마케팅에 대해 알려줘"

✅ 좋은 프롬프트:
"소규모 온라인 쇼핑몰을 운영하고 있어요.
월 예산 100만원으로 인스타그램 마케팅 전략을
초보자도 실행할 수 있게 3단계로 설명해 주세요."
```

### 프롬프트의 5가지 핵심 요소

#### 1. 역할 (Role)
AI에게 전문가 역할을 부여하세요:
```
"당신은 10년 경력의 시니어 프론트엔드 개발자입니다."
```

#### 2. 맥락 (Context)
상황과 배경을 충분히 설명하세요:
```
"React 프로젝트에서 성능 최적화를 하고 있어요.
현재 리렌더링이 너무 자주 발생하고 있어요."
```

#### 3. 지시 (Instruction)
무엇을 해야 하는지 명확하게 지시하세요:
```
"불필요한 리렌더링을 줄이는 방법을 5가지 제안해 주세요."
```

#### 4. 형식 (Format)
출력 형식을 지정하세요:
```
"각 방법마다 코드 예시를 포함하고, 난이도를 표시해 주세요."
```

#### 5. 제약 (Constraints)
제한 사항을 명시하세요:
```
"외부 라이브러리 없이, React 내장 기능만 사용해 주세요."
```

### 실전 기법

#### Few-shot 프롬프팅
예시를 함께 제공해서 원하는 패턴을 알려주는 방법이에요:

```
다음 형식으로 상품 설명을 작성해 주세요:

예시 1:
상품: 무선 이어폰
설명: 🎵 하루 종일 편안한 착용감! 12시간 배터리로
퇴근길까지 음악이 끊기지 않아요.

예시 2:
상품: 텀블러
설명: ☕ 아침에 넣은 커피가 저녁까지 따뜻하게!
진공 단열로 12시간 보온을 약속해요.

이제 작성해 주세요:
상품: 보조배터리
```

#### Chain of Thought (단계적 사고)
"단계별로 생각해 주세요"라고 요청하면 추론 능력이 향상돼요:

> 🎓 **왜 효과적인가요?**: AI도 사람처럼, 한 번에 답을 내기보다 **단계별로 풀어가면 정확도가 높아져요**. 특히 수학, 논리, 코드 디버깅처럼 추론이 필요한 작업에서 효과가 큽니다.

```
"이 수학 문제를 단계별로 풀어주세요.
각 단계에서 왜 그렇게 했는지 설명해 주세요."
```

#### 네거티브 프롬프팅
하지 말아야 할 것을 명시하는 방법이에요:

```
"전문 용어를 사용하지 마세요.
추상적인 설명 대신 구체적인 예시를 들어주세요.
'~할 수 있습니다' 대신 '~해요' 체를 사용하세요."
```

### 즉시 써먹는 프롬프트 템플릿

#### 학습용
```
[주제]를 완전 초보자에게 설명해 주세요.
일상적인 비유를 사용하고, 핵심 개념 3개를
각각 한 문장으로 요약해 주세요.
```

#### 코드 리뷰용
```
다음 코드를 리뷰해 주세요:
1. 버그가 있는지
2. 성능 개선할 부분이 있는지
3. 가독성을 높일 수 있는지
각각 구체적인 수정 방법을 코드와 함께 알려주세요.
```

#### 글쓰기용
```
[주제]에 대한 블로그 글을 작성해 주세요.
대상: [독자]
톤: [친근한/전문적인/유머러스한]
길이: [단락 수]
반드시 포함: [핵심 키워드]
```

### 흔한 실수

| 실수 | 개선 |
|------|------|
| 너무 모호하게 질문 | 구체적인 조건과 맥락 추가 |
| 한 번에 너무 많이 요청 | 단계별로 나눠서 요청 |
| 결과에 불만족하고 포기 | 프롬프트를 수정하며 반복 |
| AI 답변을 무조건 신뢰 | 중요한 정보는 반드시 검증 |

### 실전 프롬프트 비교: 같은 작업, 다른 결과

아래는 실제 업무에서 자주 쓰이는 프롬프트를 단계별로 개선한 예시예요:

#### 이메일 작성 요청
```
Level 1 (초보): "거래처에 보낼 이메일 써줘"
→ 너무 모호해서 AI가 임의로 내용을 만들어냄

Level 2 (중급): "납품 지연을 사과하는 이메일을 써줘.
상대: ABC 전자 김부장님, 사유: 원자재 수급 지연"
→ 내용은 맞지만 톤이 안 맞을 수 있음

Level 3 (고급): "다음 조건으로 비즈니스 이메일을 작성해 주세요.
- 받는 사람: ABC 전자 구매팀 김부장님
- 목적: 납품 3일 지연 사과 + 대안 제시
- 사유: 원자재 수급 차질 (구체적 날짜 언급 불필요)
- 톤: 정중하되 과도한 사과 지양, 해결 의지 강조
- 분량: 200자 내외
- 마지막에 대안 일정을 표로 정리"
→ 정확히 원하는 결과물이 나옴
```

> 💡 **핵심**: 프롬프트의 품질은 **구체성**에 비례해요. "무엇을, 누구에게, 어떤 톤으로, 어떤 형식으로"를 명시할수록 좋은 결과를 얻어요.

### 시나리오 예시

#### 예시 1: 페르소나 부여

**상황**: 블로그 글을 검수받고 싶다.

> ❌ **Before**
> "이 글 좀 봐줘."

> ✅ **After**
> "당신은 10년 경력의 테크 블로그 편집자입니다. 이 글의 논리적 흐름, 기술적 정확성, 독자 친화성을 검토하고 개선점을 3가지 제안해 주세요."

#### 예시 2: Few-shot으로 형식 지정

**상황**: 고객 리뷰를 일관된 형식으로 요약해야 한다.

> ❌ **Before**
> "이 리뷰들을 요약해줘." → AI가 매번 다른 형식으로 출력한다.

> ✅ **After**
> 예시 2개를 함께 제공한다: "리뷰: 배송 빠름, 포장 좋음 → 요약: [배송 ★5] [포장 ★5]". AI가 동일한 패턴으로 전체 리뷰를 처리한다.

#### 예시 3: Chain of Thought 활용

**상황**: 복잡한 비즈니스 의사결정을 AI에게 도움받고 싶다.

> ❌ **Before**
> "우리 회사에 Kubernetes 도입할까?" → "네, 도입하세요" 단답이 나온다.

> ✅ **After**
> "현재 인프라(EC2 3대, 트래픽 일 10만), 팀 규모(백엔드 2명), 예산을 고려해 단계별로 분석해 주세요. 도입 시 장단점과 대안도 비교해 주세요." → 구조화된 분석이 나온다.

### 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **프롬프트** | AI에게 보내는 입력 텍스트(지시·질문) | "이 코드를 리팩토링해 줘" |
| **페르소나** | AI에게 부여하는 전문가 역할 | "당신은 시니어 개발자입니다" |
| **Few-shot** | 예시를 함께 제공해 출력 패턴을 유도하는 기법 | 2~3개 입출력 예시 제공 |
| **Chain of Thought** | 단계적 사고를 유도해 추론 정확도를 높이는 기법 | "단계별로 풀어주세요" |
| **네거티브 프롬프팅** | 하지 말아야 할 것을 명시하는 기법 | "전문 용어를 사용하지 마세요" |
| **시스템 프롬프트** | AI의 행동 규칙을 정의하는 초기 설정 | 역할, 톤, 형식 규칙 지정 |

### 핵심 정리

- 프롬프트의 5요소: **역할, 맥락, 지시, 형식, 제약**
- **Few-shot**과 **Chain of Thought**가 가장 효과적인 기법이에요
- 프롬프트는 한 번에 완성되지 않아요 — **반복 개선**이 핵심이에요

> 🌟 **초보자 팁**: 프롬프트 작성이 어렵다면 이 공식으로 시작하세요 — **"[역할]으로서, [맥락] 상황에서, [형식]으로 [지시]해 주세요. 단, [제약]."** 이 틀에 내용을 채우면 꽤 좋은 결과를 얻을 수 있어요.

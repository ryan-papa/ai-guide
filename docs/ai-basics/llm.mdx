## LLM, 대규모 언어 모델이란?

LLM(Large Language Model)은 방대한 텍스트 데이터를 학습해서 인간처럼 자연스러운 언어를 이해하고 생성하는 AI 모델이에요.

### 비유로 이해하기: LLM은 만능 독서가

LLM을 **인류의 모든 책을 읽은 독서가**에 비유해 볼게요.

- 이 독서가는 수백만 권의 책, 웹페이지, 논문을 읽었어요
- 직접 경험한 것은 아니지만, 읽은 내용을 바탕으로 대부분의 질문에 답할 수 있어요
- 때로는 읽은 내용을 "섞어서" 그럴듯하지만 틀린 답을 만들기도 해요 (이것이 **환각**이에요)
- 책이 출판된 이후의 일은 모르기 때문에, 최신 정보에는 약해요 (이것이 **지식 컷오프**예요)

이 비유를 기억하면 LLM의 능력과 한계를 직관적으로 이해할 수 있어요.

### LLM의 핵심 원리

LLM의 기본 원리는 놀라울 정도로 단순해요:

> **"다음에 올 가장 적절한 단어를 예측한다"**

예를 들어:
```
"오늘 날씨가 정말 ___"
→ "좋다" (확률 35%)
→ "덥다" (확률 25%)
→ "춥다" (확률 20%)
```

이 단순한 원리가 수천억 개의 파라미터와 결합하면, 놀라운 능력이 나타나요.

#### 코드로 보는 다음 단어 예측

LLM의 핵심 원리를 수도코드로 표현하면 이래요:

```python
# LLM의 핵심 원리 (매우 단순화한 수도코드)
def generate_text(prompt, model):
    text = prompt
    while not done:
        # 1. 현재까지의 텍스트로 다음 단어의 확률 계산
        probabilities = model.predict_next_word(text)
        # 예: {"좋다": 0.35, "덥다": 0.25, "춥다": 0.20, ...}

        # 2. 확률에 기반해 다음 단어 선택
        next_word = sample(probabilities)  # "좋다" 선택

        # 3. 선택한 단어를 붙이고 반복
        text = text + next_word
    return text

# 사용 예시
result = generate_text("오늘 날씨가 정말", gpt_model)
# → "오늘 날씨가 정말 좋다. 산책하기 딱 좋은 날이다."
```

이 과정을 한 단어씩 반복하면서 긴 글이 만들어지는 거예요. 실제 LLM은 이 과정에 **Attention** 메커니즘을 사용해 문맥을 깊이 이해해요.

### LLM은 얼마나 큰가요?

| 모델 | 파라미터 수 | 학습 데이터 |
|------|-----------|------------|
| GPT-2 (2019) | 15억 개 | 40GB 텍스트 |
| GPT-3 (2020) | 1,750억 개 | 570GB 텍스트 |
| GPT-4 (2023) | 비공개 (추정 1조+) | 수 TB |
| Claude 4.6 (2025) | 비공개 | 수 TB |

파라미터가 많을수록 더 복잡한 패턴을 학습할 수 있어요.

> 💡 **파라미터란?** 뇌의 시냅스 연결 강도와 비슷해요. "고양이"라는 개념을 이해하려면 수많은 뉴런의 연결이 필요하듯, LLM도 수천억 개의 숫자(파라미터)로 언어를 표현해요. 파라미터가 많을수록 더 섬세한 언어 이해가 가능해요.

### LLM의 학습 단계

#### 1단계: 사전학습 (Pre-training)
인터넷의 방대한 텍스트를 읽으며 언어의 기본 구조를 배워요.
- 위키피디아, 책, 웹페이지 등 수조 단어를 학습
- 수천 대의 GPU로 수개월간 학습

#### 2단계: 미세조정 (Fine-tuning)
특정 작업에 맞게 추가 학습을 해요.
- 대화형으로 응답하는 방법을 학습
- 지시사항을 따르는 방법을 학습

#### 3단계: RLHF (인간 피드백 강화학습)
인간의 피드백을 받아 더 유용하고 안전한 응답을 생성하도록 개선해요.
- 사람이 "이 답변이 더 좋아요"라고 선택
- 모델이 선호되는 답변 스타일을 학습

### LLM이 할 수 있는 것들

- **대화** — 자연스럽게 질문에 답하기
- **글쓰기** — 이메일, 보고서, 시, 소설 작성
- **코딩** — 프로그래밍 코드 작성 및 디버깅
- **분석** — 문서 요약, 감성 분석, 번역
- **추론** — 수학 문제 풀기, 논리적 사고

### LLM의 한계

LLM은 강력하지만, 알아야 할 한계가 있어요:

#### 환각 (Hallucination)
사실이 아닌 내용을 마치 사실인 것처럼 자신있게 말하는 현상이에요.
- "존재하지 않는 논문을 인용"하는 것이 대표적인 예
- 왜 발생할까? LLM은 "그럴듯한 다음 단어"를 생성할 뿐, 사실 여부를 검증하는 메커니즘이 없기 때문이에요
- **대처법**: LLM의 답변을 항상 교차 검증하고, 중요한 사실은 원본 출처를 확인하세요

#### 지식 컷오프
학습 시점 이후의 정보는 알지 못해요. 최신 뉴스나 실시간 정보에 대한 답변은 정확하지 않을 수 있어요.
- **대처법**: RAG(검색 증강 생성) 등으로 최신 정보를 제공하거나, 웹 검색 기능이 있는 LLM을 활용하세요

#### 맥락 창 제한
한 번에 처리할 수 있는 텍스트의 양이 제한되어 있어요. 다만 이 한계는 점점 넓어지고 있어요.
- Claude 4.6: 1M 토큰, Gemini 2.5: 1M+ 토큰으로 점점 확장 중

### 토큰이란?

LLM은 텍스트를 **토큰(token)** 단위로 처리해요:

```
"안녕하세요" → ["안녕", "하세요"]  (2토큰)
"Hello world" → ["Hello", " world"]  (2토큰)
```

한국어는 영어보다 토큰 수가 많아지는 경향이 있어요. 이는 비용과 성능에 영향을 줘요.

> 💡 **왜 토큰이 중요할까?** LLM 서비스는 대부분 토큰 수 기준으로 요금을 부과해요. 같은 내용이라도 한국어가 영어보다 토큰 수가 1.5~2배 정도 많아져서 비용이 더 들 수 있어요. 또한 맥락 창의 크기도 토큰 단위이기 때문에, 한국어 문서는 영어 문서보다 한 번에 처리할 수 있는 양이 줄어들어요.

### 주요 LLM 생태계

현재 대표적인 LLM들을 정리하면 이래요:

| 모델 | 개발사 | 특징 |
|------|--------|------|
| **GPT-4.1** | OpenAI | 코딩·지시 추종 최적화, ChatGPT 및 API 핵심 모델 |
| **Claude 4.6 (Opus/Sonnet)** | Anthropic | 1M 맥락 창, 에이전틱 코딩에 강함, Claude Code 기반 모델 |
| **Gemini 2.5** | Google | 구글 검색 연동, 초대형 맥락 창(1M+), 멀티모달 |
| **Llama 4** | Meta | 오픈소스 MoE 아키텍처, 자유로운 커스터마이징 |
| **HyperCLOVA X** | Naver | 한국어 특화, 네이버 서비스 연동 |

### 시나리오 예시

#### 예시 1: 보고서 작성

**상황**: 주간 실적 데이터를 요약 보고서로 만들어야 한다.

> ❌ **Before (직접 작성)**
> 엑셀 데이터를 보며 1시간 동안 문장을 다듬는다. 매주 반복되는 단순 작업에 시간이 낭비된다.

> ✅ **After (LLM 활용)**
> 데이터를 붙여넣고 "주간 실적 요약 보고서를 작성해줘"라고 요청한다. 수초 내 초안이 나오고, 수정만 하면 된다.

#### 예시 2: 코드 디버깅

**상황**: 에러 메시지가 발생했지만 원인을 모르겠다.

> ❌ **Before (검색 의존)**
> 에러 메시지를 검색해 Stack Overflow를 10개 탭 열고, 내 상황에 맞는 답을 조합한다.

> ✅ **After (LLM 활용)**
> 에러 메시지와 코드를 함께 보여주면 LLM이 원인을 분석하고 수정 방안을 맥락에 맞게 제시한다.

#### 예시 3: 환각 대응

**상황**: LLM에게 최신 논문을 요약해 달라고 요청했다.

> ❌ **Before (그대로 신뢰)**
> LLM이 제시한 논문 제목과 저자를 검증 없이 인용한다. 실제로 존재하지 않는 논문일 수 있다.

> ✅ **After (검증 병행)**
> LLM의 답변을 초안으로 활용하되, 논문 제목·저자·출처를 학술 DB에서 반드시 교차 검증한다.

### 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **LLM** | 대규모 텍스트로 학습한 언어 생성 모델 | GPT-4, Claude, Gemini |
| **토큰** | LLM이 텍스트를 처리하는 최소 단위 | "안녕하세요" → 2토큰 |
| **파라미터** | 모델이 학습한 내부 가중치 수 | GPT-3: 1,750억 개 |
| **환각(Hallucination)** | 사실이 아닌 내용을 자신있게 생성하는 현상 | 존재하지 않는 논문 인용 |
| **맥락 창(Context Window)** | 한 번에 처리할 수 있는 토큰 수 제한 | 128K 토큰, 1M 토큰 |
| **미세조정(Fine-tuning)** | 사전학습된 모델을 특정 작업에 맞게 추가 학습 | 의료 상담용 모델 |
| **RLHF** | 인간 피드백으로 모델 응답 품질을 개선하는 기법 | "이 답변이 더 유용해요" 선택 |
| **프롬프트** | LLM에게 보내는 입력 텍스트(지시·질문) | "이 코드의 버그를 찾아줘" |
| **RAG** | 외부 문서를 검색해 LLM 응답에 활용하는 기법 | 최신 뉴스를 참고해 답변 생성 |
| **Attention** | 입력의 어떤 부분에 집중할지 결정하는 메커니즘 | "나는 사과를 먹었다"에서 "먹었다"가 "사과"에 주목 |
| **임베딩(Embedding)** | 단어를 숫자 벡터로 변환하는 것 | "왕"-"남자"+"여자" ≈ "여왕" |

### 핵심 정리

- LLM은 **다음 단어 예측**이라는 단순한 원리로 작동해요
- 수천억 개의 파라미터로 언어의 **복잡한 패턴**을 학습해요
- 사전학습 → 미세조정 → RLHF 순서로 점점 유용해져요
- **환각**, **지식 컷오프**, **맥락 창 제한** 등의 한계가 있어요

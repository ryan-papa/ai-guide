# 📊 머신러닝 쉽게 이해하기

머신러닝(Machine Learning)은 컴퓨터가 명시적으로 프로그래밍하지 않아도 데이터로부터 패턴을 배우는 기술이에요.

## 🍎 비유로 이해하기

아이에게 "사과"를 가르치는 상황을 생각해 볼게요:

1. 빨간 사과, 초록 사과, 노란 사과를 여러 개 보여줘요
2. 매번 "이건 사과야"라고 알려줘요
3. 아이는 점점 새로운 사과도 알아보게 돼요

머신러닝도 똑같아요! 많은 데이터(사과 사진)와 정답(사과라는 라벨)을 주면, 컴퓨터가 스스로 규칙을 찾아내요.

## 🎓 머신러닝의 세 가지 학습 방법

### 1. 지도학습 (Supervised Learning)
정답이 있는 데이터로 학습하는 방식이에요.

- **분류(Classification)**: 이메일이 스팸인지 아닌지 구분
- **회귀(Regression)**: 집값을 예측

```
데이터: [면적: 30평, 역세권, 신축] → 정답: 8억
데이터: [면적: 20평, 비역세권, 구축] → 정답: 4억
→ 새 집의 가격을 예측!
```

비유하자면 **수학 문제집의 풀이를 보며 공부하는 것**과 같아요. 문제(입력)와 정답(라벨)을 함께 보면서 풀이 방법(패턴)을 익히는 거예요.

### 2. 비지도학습 (Unsupervised Learning)
정답 없이 데이터의 패턴을 스스로 찾는 방식이에요.

- **군집화(Clustering)**: 비슷한 고객을 그룹으로 묶기
- **차원 축소**: 복잡한 데이터를 단순하게 만들기

비유하자면 **외국에서 동전을 크기·색·무게별로 분류하는 것**과 같아요. 각 동전의 가치(정답)를 모르지만, 비슷한 것끼리 그룹을 만들 수 있어요.

### 3. 강화학습 (Reinforcement Learning)
시행착오를 통해 보상을 최대화하는 방식이에요.

- 게임 AI가 이기면 보상, 지면 패널티를 받으며 전략을 개선
- AlphaGo가 바둑을 학습한 방식이 바로 강화학습이에요

## 🔄 머신러닝 학습 과정

머신러닝 모델이 학습하는 과정은 다음과 같아요:

1. **데이터 수집** — 충분한 양의 학습 데이터를 모아요
2. **데이터 전처리** — 결측치 제거, 정규화 등 데이터를 정리해요
3. **모델 선택** — 문제에 맞는 알고리즘을 선택해요
4. **학습(Training)** — 데이터를 모델에 넣고 패턴을 학습시켜요
5. **평가(Evaluation)** — 새로운 데이터로 성능을 확인해요
6. **개선** — 부족한 부분을 찾아 반복적으로 개선해요

### 간단한 코드로 보는 머신러닝

Python의 scikit-learn을 사용하면 머신러닝이 이렇게 간단해요:

```python
from sklearn.tree import DecisionTreeClassifier

# 1. 학습 데이터 준비 (특징: [키, 몸무게], 라벨: 0=고양이, 1=강아지)
X_train = [[25, 4], [30, 5], [45, 15], [50, 20]]
y_train = [0, 0, 1, 1]  # 고양이, 고양이, 강아지, 강아지

# 2. 모델 생성 및 학습
model = DecisionTreeClassifier()
model.fit(X_train, y_train)  # 데이터로부터 패턴 학습!

# 3. 새로운 데이터로 예측
new_animal = [[35, 8]]
result = model.predict(new_animal)  # 고양이? 강아지?
print("예측 결과:", "강아지" if result[0] else "고양이")
```

이 코드에서 핵심은 `model.fit()` 한 줄이에요. 이 한 줄이 데이터에서 패턴을 찾아 학습하는 과정이에요.

## ⚠️ 자주 하는 오해

| 오해 | 현실 |
|------|------|
| "ML은 만능이다" | 좋은 데이터가 없으면 좋은 모델도 없어요 |
| "데이터가 많을수록 좋다" | 양보다 **품질**이 더 중요해요 |
| "ML = AI 전체" | ML은 AI의 한 분야예요 |
| "한 번 학습하면 끝" | 환경이 바뀌면 재학습이 필요해요 |

- **병원**: X-ray 이미지를 분석해 질병을 조기에 발견해요
- **자율주행**: 도로 상황을 실시간으로 인식하고 판단해요

## 💬 시나리오 예시

### 예시 1: 집값 예측 (지도학습)

**상황**: 부동산 앱에서 매물의 적정 가격을 제시해야 한다.

> ❌ **Before (직감 기반)**
> 중개사가 경험에 의존해 "이 동네니까 대략 5억쯤"이라고 추정한다. 일관성이 없고 편향이 생긴다.

> ✅ **After (ML 적용)**
> 과거 거래 데이터(면적, 층수, 역 거리, 연식)를 학습한 모델이 조건별 예측가를 산출한다. 객관적이고 일관된 결과를 제공한다.

### 예시 2: 고객 세분화 (비지도학습)

**상황**: 쇼핑몰이 100만 고객에게 맞춤 마케팅을 하고 싶다.

> ❌ **Before (수동 분류)**
> 마케터가 "20대 여성", "30대 남성" 등 인구통계로만 분류한다. 실제 구매 패턴과 맞지 않는 경우가 많다.

> ✅ **After (ML 군집화)**
> 구매 이력·방문 패턴을 분석해 "주말 할인 사냥꾼", "신상 즉시 구매자" 등 행동 기반 그룹을 자동 발견한다.

### 예시 3: 이상 거래 탐지

**상황**: 은행이 하루 수백만 건의 거래에서 사기를 찾아야 한다.

> ❌ **Before (규칙 기반)**
> "100만 원 이상 해외 거래 시 차단" 같은 고정 규칙을 적용한다. 정상 해외 거래도 막히고, 교묘한 사기는 놓친다.

> ✅ **After (ML 탐지)**
> 각 고객의 평소 거래 패턴을 학습해 "이 거래는 이 고객답지 않다"는 이상 징후를 감지한다.

## 📖 핵심 용어

| 용어 | 설명 | 예시 |
|------|------|------|
| **머신러닝(ML)** | 데이터에서 패턴을 학습해 예측하는 기술 | 스팸 분류, 집값 예측 |
| **지도학습** | 정답 라벨이 있는 데이터로 학습 | 이메일→스팸/정상 |
| **비지도학습** | 정답 없이 데이터의 구조를 발견 | 고객 군집화 |
| **강화학습** | 보상/패널티로 최적 행동을 학습 | AlphaGo, 게임 AI |
| **과적합(Overfitting)** | 학습 데이터에만 맞고 새 데이터에 약한 상태 | 시험 족보만 외운 학생 |
| **데이터셋** | 학습·평가에 사용하는 데이터 모음 | 집값 거래 내역 10만 건 |
| **모델** | 학습된 규칙/패턴의 결과물 | 집값 예측 모델 |
| **특성(Feature)** | 모델이 학습에 사용하는 입력 변수 | 면적, 층수, 역까지 거리 |
| **라벨(Label)** | 지도학습에서 사용하는 정답 데이터 | 스팸/정상, 집값(8억) |
| **훈련/테스트 분리** | 데이터를 학습용과 평가용으로 나누는 것 | 80%로 학습, 20%로 평가 |

## 📌 핵심 정리

- 머신러닝은 **데이터에서 패턴을 찾아 예측**하는 기술이에요
- 학습 방식은 **지도학습**, **비지도학습**, **강화학습** 세 가지가 있어요
- **좋은 데이터**가 좋은 모델의 핵심이에요

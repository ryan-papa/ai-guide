## Claude Design이 뭐예요?

2026년 4월 17일 Anthropic이 공개한 **AI 디자인 에이전트**예요. 텍스트 프롬프트, 레퍼런스 이미지, 웹 캡처 같은 입력을 **클릭 가능한 인터랙티브 프로토타입**, 피치덱, 슬라이드, UI 목업으로 바꿔줘요. 엔진은 **Claude Opus 4.7**.

:::tip 한 줄 요약
"Figma를 아직 열기 전 단계" — 즉, 디자이너가 아닌 사람이 **시각 자료가 필요할 때** 쓰는 도구예요. 프로 디자인 팀의 Figma를 대체하려는 게 아니라, 창업자·PM·마케터·엔지니어가 아이디어를 눈에 보이게 만드는 게 목표예요.
:::

## 핵심 기능

| 기능 | 설명 |
|------|------|
| **텍스트 → 프로토타입** | 자연어 설명만으로 인터랙티브 UI 생성 |
| **정적 → 인터랙티브** | 스크린샷·시안을 업로드하면 동작 로직을 자동으로 추가 |
| **브랜드 시스템 자동 인식** | 코드베이스·Figma·디자인 파일을 읽고 색상·타이포·컴포넌트 패턴을 추출해 일관성 유지 |
| **웹 캡처 도구** | 실제 사이트에서 시각 요소를 끌어와 리얼한 목업 구성 |
| **프런티어 디자인** | 3D 비주얼, 보이스 인터페이스, 모션 효과 지원 |
| **멀티플레이어 협업** | 여러 팀원이 동시에 편집 |
| **Claude Code 핸드오프** | 프로토타입을 그대로 Claude Code로 넘겨 프로덕션 코드로 변환 |
| **다양한 내보내기** | ZIP, PDF, PPTX, HTML, 공유 URL |

## 워크플로우

1. **온보딩** — 코드베이스, Figma 파일, 브랜드 에셋을 읽혀서 디자인 시스템을 학습시켜요.
2. **시작** — 텍스트 프롬프트, 레퍼런스 이미지, 문서, 웹 캡처 중 하나로 프로젝트를 열어요.
3. **다듬기** — 대화형 채팅, 인라인 코멘트, 직접 편집으로 다듬어요.
4. **내보내기** — ZIP/PDF/PPTX/HTML로 뽑거나, **Claude Code로 바로 넘겨** 코드로 변환해요.

:::info Claude Code와의 닫힌 루프
Claude Design → Claude Code로 바로 이어지는 게 이 제품의 핵심 강점이에요. "PM이 프로토타입 → 엔지니어가 스펙 해석 → 구현" 이라는 전통적인 단계를 건너뛰고, 프로토타입 자체가 코드 생성의 입력이 돼요.
:::

## 언제 쓰면 좋아요?

- **창업자·PM**: 투자자 미팅 전날 밤 피치덱·데모 목업을 빠르게 준비
- **마케터**: 랜딩 페이지·캠페인 시안을 디자이너 리소스 없이 시안화
- **엔지니어**: 백엔드/인프라 개발자가 간단한 내부 도구 UI를 직접 디자인
- **디자이너**: 초기 탐색 단계에서 다양한 방향을 빠르게 실험

## 실전 예시

### 예시 1 — SaaS 랜딩 페이지 초안 (PM)

입력 프롬프트:

> "B2B 협업 노트 앱의 랜딩 페이지를 만들어줘. 히어로 섹션에는 한 문장 헤드라인 + CTA 버튼, 그 아래 3-컬럼 기능 소개, 하단에 요금제 표 3개(Free/Pro/Team). 톤은 Linear처럼 미니멀·다크 모드 기본."

결과:
- 클릭 가능한 HTML 프로토타입이 즉시 생성 — CTA 버튼을 누르면 회원가입 모달이 열리고, 요금제 카드는 hover 애니메이션이 들어가 있어요.
- "요금제 표 가운데 Pro에 Most Popular 배지 추가해줘" 한 마디면 인라인으로 수정돼요.
- **Export to Claude Code**를 누르면 Next.js + Tailwind 프로젝트로 떨어져요.

:::tip 팁
기존 서비스 링크를 **웹 캡처**로 넣으면 색감·폰트·여백 패턴을 그대로 따와서 "자사 브랜드톤 유지한 리뉴얼"에 좋아요.
:::

### 예시 2 — 내부 대시보드 목업 (엔지니어)

입력:

> "우리 코드베이스 (첨부한 zip) 의 컴포넌트 라이브러리를 보고, 데이터 파이프라인 상태 대시보드를 만들어줘. 상단에 지표 4개 (처리량, 지연, 실패율, 대기 큐), 중앙에 타임라인 차트, 우측에 최근 알림 리스트."

결과:
- 첨부한 코드베이스의 `Button`, `Card`, `Badge` 컴포넌트를 **그대로 인식**해서 동일한 디자인 토큰으로 구성해요.
- 차트는 더미 데이터로 동작하는 인터랙티브 프로토타입.
- Claude Code 핸드오프 시 **우리 프로젝트 구조에 맞춰** 컴포넌트 경로와 import까지 맞춰 생성돼요.

### 예시 3 — 투자자용 피치덱 (창업자)

입력:

> "Seed 라운드용 10슬라이드 피치덱. 문제 → 솔루션 → 시장 → 제품 데모 → 트랙션 → 팀 → 재무 → 요청 → 로드맵 → 감사. 섹션별로 내 노션 문서(첨부)에서 근거 문장 뽑아서 써줘."

결과:
- 10장짜리 인터랙티브 슬라이드 (스크롤·클릭 전환).
- 제품 데모 슬라이드는 **별도 클릭 가능한 프로토타입**이 임베드 돼요 — 스크린샷 아닌 실제 동작.
- `.pptx` 로 내보내면 오프라인 발표용으로도 쓸 수 있어요.

### 예시 4 — A/B 시안 동시 탐색 (디자이너)

입력:

> "체크아웃 페이지 시안 3개. (A) 원페이지, (B) 3단계 스텝, (C) 모달 오버레이. 셋 다 같은 브랜드 시스템 사용."

결과:
- **멀티플레이어 모드**로 팀원을 초대해 실시간 코멘트·편집.
- 각 시안은 독립된 프로토타입이라 사용자 테스트 링크를 그대로 공유 가능.

## 프롬프트 작성 팁

| 안 좋은 예 | 좋은 예 |
|------------|---------|
| "예쁜 대시보드 만들어줘" | "다크 모드 기본, 지표 4개(상단) + 타임라인 차트(중앙) + 알림 리스트(우측) 구조의 SRE 대시보드" |
| "세련되게" | "Linear/Vercel 톤, 미니멀, 둥근 모서리 8px, 주 색상은 #5B8DEF" |
| "로그인 페이지" | "이메일 + 비밀번호 + SSO 3개(Google/GitHub/SSO) 버튼, 오른쪽엔 제품 스크린샷, 모바일에서는 스크린샷 숨김" |

:::info 핵심
**구조 · 톤 · 인터랙션**을 구체적으로 적을수록 첫 결과 품질이 좋아요. 이후 대화형으로 다듬는 건 덤.
:::
